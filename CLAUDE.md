This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Project Overview

DeepScript is a **Transcript Intelligence Engine** — a separate package from AudioScript. AudioScript handles audio-to-text transcription; DeepScript performs text-to-intelligence analysis on transcripts from any source (AudioScript, Zoom, Otter, manual).

**Status:** Feature-complete. PRD in `PRD-deepscript.md`. 165 tests passing.

## Architecture

DeepScript follows a classification-driven pipeline:

```
Transcript JSON → Classifier → Type-specific Analyzer → Formatted Output
                                      ↓
                              CMS Episode (optional)
```

### Module Structure

- **`core/`** — `classifier` (LLM-first with keyword fallback), `topic_segmenter` (rule/LLM/hybrid), `communication` (talk ratio, engagement), `tagger` (MiNotes/Obsidian frontmatter)
- **`analyzers/`** — Auto-discovered via registry. 15 modules, 50+ call types. Adding a new analyzer = create file with `supported_types` — no other edits needed.
  - Tier 1: `business`, `sales` (MEDDIC/BANT/SPIN), `discovery` (Mom Test/JTBD), `support`, `qbr`, `interview` (STAR)
  - Tier 2: `pmf` (8-dimension Ellis/Vohra), `pitch`, `recruiting`, `management`, `operations`, `customer`, `education`
  - Tier 3: `relationship` (Gottman/NVC, opt-in), `specialized` (podcast/therapy/simple)
- **`llm/`** — 7 providers (Claude, OpenAI, Ollama, vLLM, SGLang, NIM + custom base_url). Retry with exponential backoff. Async support. Combined analysis prompts (1 LLM call instead of 3-5). Budget enforcement. 20 prompt templates.
- **`formatters/`** — JSON + Markdown with type-specific section rendering (PMF dimensions, STAR scores, signals, etc.)
- **`cli/`** — `analyze` (single + batch), `classify`, `usage`, `playbook`, `dashboard`, `prep`
- **`cms_bridge/`** — CallEpisode model + JSONL writer, playbook generation from episode patterns, PMF dashboard (Vohra engine), working memory assembly for call prep
- **`integrations/`** — Calendar context (ms365-cli/gwscli), notification routing
- **`utils/`** — Processing manifest (SHA-256 dedup), logging
- **`mcp_server.py`** — FastMCP server for agent integration

### Key Design Principles

- **Hybrid analysis**: Rule-based (free, fast) + LLM-enhanced (accurate, costs money). Every feature should degrade gracefully without LLM access (`llm.provider: none`).
- **Classification drives analysis**: Auto-detect call type, then run the appropriate analyzer. Each analyzer produces structured sections specific to that call type.
- **Privacy-first for relationship analysis**: Opt-in only, private by default, positive framing, research-backed (Gottman, NVC). Never runs automatically.
- **Input agnostic**: Accepts any transcript as JSON with `text`, `segments[]`, and optional `diarization` data. AudioScript transcripts include `speaker_cluster_id` for cross-call identity tracking.

### Cross-Project Integration

- **BFlow** (`/root/projects/BTask/packages/bflow/`): Orchestrator — DeepScript is a BFlow concierge skill (`SKILL.md` + CLI). BFlow handles discovery, scheduling, and composition with other tools. No complex YAML workflows — each step is a CLI invocation.
- **AudioScript** (`/root/projects/audioscript/`): Upstream — produces transcripts that DeepScript analyzes. Can invoke DeepScript as post-processing via `.audioscript.yaml` sync config.
- **ms365-cli** (`/root/projects/ms365-cli/`): Calendar context enrichment (meeting attendees, subject, agenda) and notification routing (Teams, email). Used via CLI commands.
- **gwscli** (`/root/projects/gwscli/`): Google Workspace equivalent — Calendar context and Google Chat notifications. Used via CLI commands.
- **BTask CMS** (`/root/projects/BTask/packages/cms/`): Downstream — CallEpisodes stored as episodic memory, distiller generates playbooks from patterns across calls.
- **MiNotes**: Transcript pages with tags/properties generated by the tagger module.

### Config

Configuration via `.deepscript.yaml` with sections for: classification, topics, communication, business/sales/discovery/relationship analysis, LLM provider settings, and output formatting. See PRD section 9 for full schema.

## Workflow

### Plan First
- Enter plan mode for any non-trivial task (3+ steps or architectural decisions)
- If something goes sideways, stop and re-plan immediately — don't keep pushing
- Use plan mode for verification steps, not just building
- Write detailed specs upfront to reduce ambiguity

### Subagents
- Use subagents liberally to keep main context window clean
- Offload research, exploration, and parallel analysis to subagents
- For complex problems, throw more compute at it via subagents
- One task per subagent for focused execution

### Task Management
- **Plan**: Write plan to `tasks/todo.md` with checkable items
- **Verify**: Check in before starting implementation
- **Track**: Mark items complete as you go, explain changes at each step
- **Document**: Add review section to `tasks/todo.md`
- **Learn**: Update `tasks/lessons.md` after any correction — write rules that prevent the same mistake, review at session start

### Verification Before Done
- Never mark a task complete without proving it works
- Diff behavior between main and your changes when relevant
- Run tests, check logs, demonstrate correctness
- Ask: "Would a staff engineer approve this?"

### Demand Elegance (Balanced)
- For non-trivial changes: pause and ask "is there a more elegant way?"
- If a fix feels hacky: step back and implement the elegant solution
- Skip this for simple, obvious fixes — don't over-engineer

### Autonomous Bug Fixing
- When given a bug report: just fix it. Don't ask for hand-holding
- Point at logs, errors, failing tests — then resolve them
- Find root causes. No temporary fixes. Minimal impact — only touch what's necessary.

## Build & Test

```bash
pip install -e ".[dev]"              # Install with dev deps
pip install -e ".[dev,llm]"          # Install with LLM support (anthropic)
pytest                                # Run all tests (165 tests)
pytest tests/test_pmf_analyzer.py    # Run single test file
pytest -k "test_classify"            # Run tests matching pattern
deepscript --version                 # Verify CLI
deepscript analyze file.json         # Analyze a transcript
deepscript analyze dir/ -r --new-only  # Batch with dedup
```

## Tech Stack

- **Language:** Python 3.11+
- **CLI:** Typer (matching AudioScript's patterns)
- **Config:** Pydantic + PyYAML
- **LLM:** Claude/OpenAI/Ollama via provider abstraction
- **Dependencies:** Lightweight (~50MB) — LLM APIs, regex, nltk. No torch/GPU required.
- **Install:** `pip install deepscript` (standalone) or `pip install audioscript[deepscript]` (full pipeline)
